Hur och varför jag återställde en commit.

1. Gjorde ett nytt recept (fryspizza.txt) och la till instruktioner. 
2. Comittade (5b8ee15).
3. Ersatte jag alla instruktioner i fryspizza.txt med massa random text
4. Comittade (eb07019).
5. Skrev: git reset --hard eb07019
6. Alla mina instruktioner i fryspizza.txt blev återställda


(NEDANSTÅENDE HAR JAG ÅTERSKAPAT MED EN PATCHFIL, jag skrev detta, gjorde git add, gjorde git commit, tog bort allt, återskapade med hjälp av git am patches/namnpåpatchfil.patch)
Jag råkade ta bort en commit för mycket med "git reset --hard HEAD~1", detta gjorde att även min patchfil försvann. Jag lyckedes göra en ny patchfil från rflog eftersom commiten som patchfilen representerade låg kvar där. Jag gjorde detta  med git format-patch -1 6b48d00 -o patches/ efter att ja gjort "mkdir patches" igen.

Vad skillnaden är mellan "git am" och "git apply".

- git apply patches/namnpåpatchfil.patch applicerar ändringarna som finns i patchfilen men skapar ingen "git add" eller "git commit", du måste själva göra detta.

- git am patches/namnpåpatchfil.patch (applicerar ändringarna som finns i patchfilen, gör en git add, tar med patchfilens commitmeddelande och metadata, gör en ny commit)
